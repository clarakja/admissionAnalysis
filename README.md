# 🎓 입시 홍보 분석 대시보드

최근 3개년 지원자 데이터를 분석해 입시 홍보 전략 수립을 돕는 Streamlit 대시보드.

## 📁 파일 구성 (2개)

| 파일 | 역할 | 크기 |
|---|---|---|
| **analysis_engine.py** | 데이터 로드 + 분석 함수 (순수 로직, UI 없음) | ~650줄 |
| **app.py** | Streamlit UI — 사이드바·8개 페이지·저장소 | ~1,500줄 |
| requirements.txt | 패키지 목록 | — |

두 파일을 **같은 폴더**에 두고 실행하면 됩니다.

## 🚀 실행 — 두 줄이면 끝

```bash
pip install -r requirements.txt
streamlit run app.py
```

브라우저가 자동으로 열립니다 (http://localhost:8501).

> Python **3.10 이상** 필요.

## 🌐 Streamlit Cloud 배포 (영구 URL)

1. GitHub 에 리포지토리 만들고 **3개 파일**(`app.py`, `analysis_engine.py`, `requirements.txt`)을 푸시
2. [share.streamlit.io](https://share.streamlit.io) → New app → 리포 선택
3. Main file path: `app.py` → Deploy
4. 2분 뒤 고유 URL 생성 완료

## 📊 서사 구조

```
🏠 홈              한눈에 보는 현재 상태
  ↓
📊 현황 진단       "우리 대학은 지금 어떤가?"        (6개)
🎓 학과 진단       "학과별로 어떻게 다른가?"         (6개)
🏫 피더 스쿨       "어디에서 지원자가 오는가?"       (7개)
  ↓
🎯 전략 발굴 ⭐   "어디에 집중해야 할까?"           (7개 - 핵심)
🔽 Funnel          "단계별 어디서 이탈하는가?"       (5개)
  ↓
🔍 심층 분석       "특정 대상을 자세히 보기"         (3개)
  ↓
📝 공유·리포트     "발견을 공유하자"                (2개)
```

총 **38개 분석** + 홈 대시보드.

## 🎯 전략 발굴 — 핵심 도구: 4분류 매트릭스

| 분류 | 의미 | 전략 |
|---|---|---|
| A: 지원강·등록강 | 핵심관리 | 관계 유지·심화 |
| **B: 지원강·등록약** | 전환율개선 | **왜 이탈하는지 조사** |
| **C: 지원약·등록강** | 잠재력발굴 | **지원 늘리면 바로 효과** |
| D: 지원약·등록약 | 신규개척 | 장기 관계 구축 |

산점도에서 중앙값 기준선을 기준으로 4분면에 배치되며, 호버하면 학교별 상세 확인 가능.
